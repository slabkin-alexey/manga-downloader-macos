Manga Downloader (macOS) - Documentation

Version: 1.0 | Platform: native SwiftUI app for macOS

What It Is

Manga Downloader is a desktop app that downloads manga from supported websites and converts content into EPUB optimized for e-ink reading. Source websites: manga/in/ua (manga.in.ua) and com-x.life (com-x.life).

Features

  • Service picker with two sources: manga/in/ua and com-x.life.
  • Input as a single URL or a queue of URLs (comma/newline separated).
  • manga/in/ua mode auto-switch: URL input for download, text input for search.
  • Chapter/volume filtering with range syntax: 1-2, 1-2,4,10-12.
  • Optional custom EPUB cover image.
  • Per-volume output: one EPUB per volume.
  • Real-time localized logs with clickable links and multi-level progress bars.
  • Start/Stop control with cancellation and process cleanup.
  • Turbo mode (performance profile switch) with warning and persisted state.
  • Completion alert + macOS system notification.
  • Localization: English, Ukrainian, Russian.

Supported Services

manga/in/ua

  • Search flow: query website, parse cards, print readable results in logs.
  • Download flow: resolve links, fetch chapter list, apply filters, download images, build CBZ and EPUB.
  • EPUB page ordering follows manga-style reading semantics used by the app.

com-x.life

  • URL download flow with metadata/chapter parsing from site payloads.
  • Archive download and extraction, then conversion and packaging.
  • Auth flow with cookie persistence, in-app login WebView, retry logic, and reset-login action.
  • Parsing robustness for payload variants and chapter ID field variants.

How It Works (Step-by-Step)

  1. Select source service on startup screen.
  2. Provide URL(s) or search text (manga/in/ua only).
  3. Optionally set chapter/volume filters and output directory.
  4. Optionally select a custom cover image.
  5. Start processing. For each chapter/page set, pipeline is executed:
  • Download original image/archive.
  • Convert to grayscale (B/W).
  • Optimize for e-ink (max height 1080px, keep aspect ratio, no upscaling).
  • Convert to HEIC.
  • Remove metadata/EXIF where possible.
  • Stage by volume and package artifacts.
  • Create CBZ then EPUB per volume, then clean temporary intermediates on success.

Authentication Flow (com-x.life)

  1. Downloader detects that authorization is required.
  2. If valid cookies exist, flow retries without login UI.
  3. If not, app opens in-app login WebView and captures cookies.
  4. Cookies are persisted and download flow is retried.
  5. User can reset saved login cookies from feature UI.

Filtering Rules

  • Accepted syntax: mixed values and ranges (example: 1-2,4,10-12).
  • If both filters are provided, chapter filter is applied within matching volumes.
  • Invalid syntax returns explicit validation error.

Progress, Logging, and Process Control

  • Log panel shows event-oriented text and clickable links.
  • Progress stages include downloading, B/W conversion, resolution optimization, HEIC optimization, and volume EPUB conversion.
  • Multi-level progress tracks pages in chapter, chapters in volume, and overall volume process.
  • Stop action cancels task, terminates registered shell processes, logs cancellation, and restores fan mode to auto if used.

Turbo Mode and Performance

  • Turbo OFF: background priorities, lower parallelism/network aggressiveness, nice-wrapped shell commands.
  • Turbo ON: high priorities, higher parallelism based on CPU cores, higher network responsiveness.
  • Enabling turbo shows warning alert; state is persisted globally.
  • Optional fan helper integration: max fan mode during active turbo processing, auto mode restored on stop/completion.

Dependencies and Tooling

  • Image conversion prefers ImageMagick magick when available.
  • Fallback tooling uses native macOS commands like sips.
  • App can check for missing magick, run Homebrew install command in Terminal, and poll for completion.
  • TelegramReporter integration sends startup telemetry/log event using configured token/chat/app tag.

Usage Guide

  1. Launch app and choose manga/in/ua or com-x.life.
  2. Paste one or many links. For manga/in/ua search, enter plain text instead of URL.
  3. Set chapter/volume filters if needed.
  4. Select output directory and optional EPUB cover image.
  5. Optionally enable Turbo mode (warning appears).
  6. Press Start and monitor logs/progress.
  7. When done, review completion alert/notification and generated EPUB files by volume.

FAQ / Troubleshooting

Input is rejected

Empty input is blocked. In com-x.life mode all queue entries must be valid URLs.

Filter does not apply

Check syntax and remove invalid tokens. Use only numbers and ranges separated by commas.

com-x.life asks for authorization

Complete login in WebView. If old cookies are stale, use reset-login and retry.

Image conversion quality/speed issues

Install ImageMagick (magick) for preferred conversion path. Otherwise app falls back to native tools.

High CPU/temperature

Disable Turbo mode for lower system pressure. Turbo mode intentionally increases parallelism and responsiveness.
