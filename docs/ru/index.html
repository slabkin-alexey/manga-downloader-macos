Manga Downloader (macOS) - Документация

Версия: 1.0 | Платформа: нативное SwiftUI-приложение для macOS

Что это

Manga Downloader - это настольное приложение, которое загружает мангу с поддерживаемых сайтов и конвертирует контент в EPUB, оптимизированный для e-ink. Поддерживаемые источники: manga/in/ua (manga.in.ua) и com-x.life (com-x.life).

Интерфейс приложения поддерживает три языка: украинский, английский и русский.

Возможности

  • Выбор сервиса с двумя источниками: manga/in/ua и com-x.life.
  • Ввод одного URL или очереди URL (через запятую/перенос строки).
  • Автопереключение режима manga/in/ua: URL для загрузки, текст для поиска.
  • Фильтрация глав/томов с синтаксисом диапазонов: 1-2, 1-2,4,10-12.
  • Опциональная пользовательская обложка EPUB.
  • Вывод по томам: один EPUB на том.
  • Локализованные логи в реальном времени с кликабельными ссылками и многоуровневым прогрессом.
  • Управление Start/Stop с отменой и очисткой процессов.
  • Turbo-режим (профиль производительности) с предупреждением и сохранением состояния.
  • Завершение: alert в UI + системное уведомление macOS.
  • Локализация: English, Ukrainian, Russian.

Поддерживаемые сервисы

manga/in/ua

  • Поиск: запрос к сайту, парсинг карточек, вывод читаемых результатов в логах.
  • Загрузка: резолв ссылок, получение списка глав, применение фильтров, загрузка изображений, сборка CBZ и EPUB.
  • Порядок страниц EPUB формируется по manga-style логике, используемой приложением.

com-x.life

  • URL-загрузка с парсингом метаданных и таблицы/API глав сайта.
  • Загрузка архивов глав, распаковка, затем конвертация и упаковка.
  • Auth-поток с сохранением cookies, in-app login WebView, повторной попыткой и действием reset login.
  • Устойчивый парсинг для вариантов payload и разных названий полей chapter ID.

Как это работает (пошагово)

  1. Выберите сервис на стартовом экране.
  2. Укажите URL(ы) или текст поиска (только для manga/in/ua).
  3. При необходимости задайте фильтры глав/томов и директорию вывода.
  4. При необходимости выберите пользовательскую обложку.
  5. Запустите процесс. Для каждого набора глава/страницы выполняется pipeline:
  • Загрузка исходного изображения/архива.
  • Конвертация в grayscale (B/W).
  • Оптимизация под e-ink (макс. высота 1080px, сохранение пропорций, без апскейла).
  • Конвертация в HEIC.
  • Удаление metadata/EXIF, где возможно.
  • Промежуточная сборка по томам и упаковка.
  • Создание CBZ, затем EPUB по каждому тому, и очистка временных артефактов после успеха.

Auth-поток (com-x.life)

  1. Downloader определяет, что требуется авторизация.
  2. Если валидные cookies уже есть, поток повторяется без UI логина.
  3. Если нет, открывается in-app login WebView и cookies считываются.
  4. Cookies сохраняются, после чего поток загрузки запускается повторно.
  5. Пользователь может сбросить сохраненные cookies через reset login.

Правила фильтрации

  • Поддерживаемый синтаксис: отдельные значения и диапазоны (например, 1-2,4,10-12).
  • Если заданы оба фильтра, фильтр глав применяется внутри отфильтрованных томов.
  • Невалидный синтаксис возвращает явную ошибку валидации.

Прогресс, логи и управление процессом

  • Панель логов показывает события в читаемом виде и кликабельные ссылки.
  • Этапы прогресса: загрузка, B/W-конвертация, оптимизация разрешения, HEIC-оптимизация, EPUB-конвертация тома.
  • Многоуровневый прогресс отслеживает страницы главы, главы в томе и общий прогресс по томам.
  • Stop отменяет задачу, завершает зарегистрированные shell-процессы, логирует отмену и возвращает режим вентиляторов в auto (если использовался).

Turbo-режим и производительность

  • Turbo OFF: фоновые приоритеты, меньший параллелизм/сетевая агрессивность, nice для shell-команд.
  • Turbo ON: высокие приоритеты, больший параллелизм по числу CPU-ядер, более высокая сетевая отзывчивость.
  • При включении Turbo показывается предупреждение; состояние сохраняется глобально.
  • Опциональная интеграция fan helper: max-режим во время активной обработки в Turbo, возврат в auto при остановке/завершении.

Зависимости и инструменты

  • Для лучшего качества конвертации приоритетно используется ImageMagick magick.
  • Fallback-инструменты: нативные команды macOS, включая sips.
  • Приложение может проверять наличие magick, запускать команду установки через Homebrew в Terminal и опрашивать статус завершения.
  • Интеграция TelegramReporter отправляет стартовую телеметрию/лог с настроенными token/chat/app tag.

Руководство по использованию

  1. Запустите приложение и выберите manga/in/ua или com-x.life.
  2. Вставьте одну или несколько ссылок. Для поиска в manga/in/ua введите обычный текст вместо URL.
  3. При необходимости настройте фильтры глав/томов.
  4. Выберите директорию вывода и опциональную обложку EPUB.
  5. При необходимости включите Turbo (с предупреждением).
  6. Нажмите Start и отслеживайте логи/прогресс.
  7. После завершения проверьте alert/уведомление и EPUB-файлы по томам.

FAQ / Устранение неполадок

Ввод отклоняется

Пустой ввод блокируется. В режиме com-x.life все элементы очереди должны быть валидными URL.

Фильтр не применяется

Проверьте синтаксис и удалите невалидные токены. Используйте только числа и диапазоны через запятую.

com-x.life требует авторизацию

Выполните вход в WebView. Если cookies устарели, используйте reset login и повторите запуск.

Проблемы с качеством/скоростью конвертации

Установите ImageMagick (magick) для предпочтительного пути конвертации. Иначе приложение использует нативные инструменты.

Высокая нагрузка CPU/температура

Отключите Turbo для снижения нагрузки. Turbo намеренно увеличивает параллелизм и скорость обработки.

Конфиденциальность и юридическое предупреждение

  • Приложение обрабатывает URL и загружаемый контент, предоставленные пользователем.
  • Cookies авторизации com-x.life сохраняются для повторных авторизованных запросов и могут быть сброшены пользователем.
  • Стартовая телеметрия/логи через TelegramReporter работают только при наличии настроенных учетных данных.
  • Пользователь несет ответственность за соблюдение условий сервисов и применимого авторского права.
